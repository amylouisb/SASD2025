# ตัวอย่าง C# Class 3 ประเภทจาก .NET API

## Utility Class  
### System.Convert

`System.Convert` เป็นคลาสที่ใช้สำหรับแปลงชนิดข้อมูล เช่น แปลง string เป็น int หรือ double เป็น string เป็นต้น

เหตุผลที่จัด `System.Convert` เป็น **Utility Class** คือ  
คลาสนี้มีแต่เมธอดแบบ `static` ทั้งหมด และไม่จำเป็นต้องสร้าง object ขึ้นมาเพื่อใช้งาน เราสามารถเรียกใช้เมธอดต่าง ๆ ได้โดยตรงผ่านชื่อคลาส เช่น `Convert.ToInt32()` หรือ `Convert.ToString()`  

นอกจากนี้ `System.Convert` ไม่ได้มี state ภายในของตัวเอง และไม่ต้องเก็บข้อมูลใด ๆ ระหว่างการใช้งาน หน้าที่ของคลาสคือรับ input เข้ามา แปลงค่า แล้วส่ง output ออกมาเพียงอย่างเดียว จึงเหมาะกับการเป็น Utility Class ตามนิยามในเชิง Object-Oriented Design

## Mutable Class  
### System.IO.MemoryStream

`System.IO.MemoryStream` เป็นคลาสที่ใช้จัดการข้อมูลแบบ stream ที่ถูกเก็บไว้ในหน่วยความจำ (RAM) แทนที่จะเป็นไฟล์บนดิสก์

เหตุผลที่จัด `MemoryStream` เป็น **Mutable Class** คือ  
หลังจากสร้าง object ขึ้นมาแล้ว state ภายในของ object สามารถเปลี่ยนแปลงได้ตลอดเวลา เช่น  
- การเขียนข้อมูลเพิ่มเข้าไปใน stream  
- การอ่านข้อมูลซึ่งทำให้ตำแหน่ง (position) ภายใน stream เปลี่ยนไป  
- ขนาดของข้อมูล (length) เปลี่ยนตามการเขียนหรืออ่าน  

การเรียกใช้เมธอดต่าง ๆ อย่าง `Write()` หรือ `Read()` จะทำให้ข้อมูลภายใน object เปลี่ยนแปลงโดยตรง ซึ่งตรงกับลักษณะของ mutable object ที่ state สามารถเปลี่ยนไปตามการใช้งานได้

## Immutable Class  
### System.Uri

`System.Uri` ใช้แทนที่อยู่ของทรัพยากร เช่น URL หรือ URI ต่าง ๆ บนอินเทอร์เน็ต

เหตุผลที่จัด `System.Uri` เป็น **Immutable Class** คือ  
เมื่อสร้าง `Uri` object ขึ้นมาแล้ว ค่า URI ภายใน object จะไม่สามารถเปลี่ยนแปลงได้ ไม่มี setter หรือเมธอดใดที่แก้ไขค่าเดิมของ URI โดยตรง หากต้องการเปลี่ยน URL จะต้องสร้าง `Uri` object ใหม่ขึ้นมาแทน  

ลักษณะนี้ทำให้ `Uri` มีความปลอดภัยในการใช้งานร่วมกันหลายส่วนของโปรแกรม และลดปัญหาการเปลี่ยนค่าโดยไม่ตั้งใจ ซึ่งเป็นคุณสมบัติสำคัญของ immutable object
